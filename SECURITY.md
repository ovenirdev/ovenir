## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to: **security@ovenir.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### What to Expect

- **Acknowledgment**: Within 48 hours
- **Status update**: Within 7 days
- **Resolution**: Depends on severity, typically within 30 days

### Safe Harbor

We consider security research conducted in good faith to be authorized. We will not pursue legal action if you:

- Act in good faith
- Avoid privacy violations, data destruction, or service disruption
- Report vulnerabilities promptly
- Do not publicly disclose until we've addressed the issue

## Security Best Practices for Users

1. **Paste Guard**: Enable Paste Guard to detect secrets before pasting
2. **Offline Mode**: For sensitive operations, use OVENIR offline
3. **Clear Data**: Use the panic clear feature after handling sensitive data
4. **Verify Source**: Only use OVENIR from official sources (ovenir.com)

## Scope

In scope:
- ovenir.com and subdomains
- Official OVENIR repositories
- Client-side security issues

Out of scope:
- Third-party services
- Social engineering attacks
- Physical attacks
